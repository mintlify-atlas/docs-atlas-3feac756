## Overview

This guide provides comprehensive installation instructions for the Fraud Detection ML System. Follow these steps to set up your development environment and get the system running on your machine.

<Note>
  Installation typically takes 10-15 minutes depending on your internet connection and system specifications.
</Note>

---

## System Requirements

### Hardware Requirements

| Component | Minimum | Recommended |
|-----------|---------|-------------|
| **CPU** | 2 cores | 4+ cores |
| **RAM** | 4 GB | 8+ GB |
| **Storage** | 2 GB free | 5+ GB free |
| **OS** | Linux, macOS, Windows 10+ | Linux (Ubuntu 18.04+) |

### Software Prerequisites

Before installing, ensure you have the following software:

<AccordionGroup>
  <Accordion title="Python 3.8 (Required)">

## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    First, clone the fraud detection repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd source
    ```
    
    **Verify directory structure:**
    ```bash
    ls -la
    ```
    
    You should see directories like:
    - `Training_Batch_Files/`
    - `Prediction_Batch_files/`
    - `application_logging/`
    - Files: `main.py`, `trainingModel.py`, `requirements.txt`
  </Step>

  <Step title="Create Conda Environment">
    Create a new conda environment named `fraud-detection` with Python 3.8:
    
    ```bash
    conda create -n fraud-detection python=3.8 -y
    ```
    
    This command:
    - Creates an isolated environment named `fraud-detection`
    - Installs Python 3.8
    - `-y` flag automatically confirms the installation
    
    **Expected output:**
    ```
    Collecting package metadata (current_repodata.json): done
    Solving environment: done
    
    ## Package Plan ##
    
      environment location: /home/user/miniconda3/envs/fraud-detection
      
      added / updated specs:
        - python=3.8
    
    ...
    
    # To activate this environment, use
    #
    #     $ conda activate fraud-detection
    ```
  </Step>

  <Step title="Activate the Environment">
    Activate the newly created conda environment:
    
    ```bash
    conda activate fraud-detection
    ```
    
    Your terminal prompt should change to show `(fraud-detection)` at the beginning:
    
    ```bash
    (fraud-detection) user@machine:~/source$
    ```
    
    <Warning>
      You must activate this environment every time you open a new terminal session before working with the fraud detection system.
    </Warning>
    
    **Tip:** To automatically activate the environment, add this to your `.bashrc` or `.zshrc`:
    ```bash
    # Auto-activate fraud-detection environment
    conda activate fraud-detection
    ```

  </Step>

## Platform-Specific Notes

<Tabs>
  <Tab title="Linux">
    ### Linux Installation Notes
    
    Linux is the recommended platform for production deployments.
    
    **Additional dependencies for Ubuntu/Debian:**
    ```bash
    sudo apt-get update
    sudo apt-get install -y build-essential python3-dev
    ```
    
    **For XGBoost compilation (if needed):**
    ```bash
    sudo apt-get install -y cmake libboost-all-dev
    ```
    
    **Running as a service:**
    
    Create `/etc/systemd/system/fraud-detection.service`:
    ```ini
    [Unit]
    Description=Fraud Detection ML System
    After=network.target
    
    [Service]
    Type=simple
    User=your-user
    WorkingDirectory=/path/to/source
    Environment="PATH=/home/your-user/miniconda3/envs/fraud-detection/bin"
    ExecStart=/home/your-user/miniconda3/envs/fraud-detection/bin/python main.py
    Restart=on-failure
    
    [Install]
    WantedBy=multi-user.target
    ```
    
    Enable and start:
    ```bash
    sudo systemctl enable fraud-detection
    sudo systemctl start fraud-detection
    ```
  </Tab>
  
  <Tab title="macOS">
    ### macOS Installation Notes
    
    **Install Xcode Command Line Tools:**
    ```bash
    xcode-select --install
    ```
    
    **Using Homebrew (recommended):**
    ```bash
    brew install python@3.8 git
    ```
    
    **M1/M2 Mac considerations:**
    
    If you're on Apple Silicon, some packages may require Rosetta 2:
    ```bash
    # Install Rosetta 2
    softwareupdate --install-rosetta
    
    # Create environment with x86_64 architecture
    CONDA_SUBDIR=osx-64 conda create -n fraud-detection python=3.8 -y
    conda activate fraud-detection
    conda config --env --set subdir osx-64
    ```
  </Tab>
  
  <Tab title="Windows">
    ### Windows Installation Notes
    
    **Use Anaconda Prompt or PowerShell:**
    
    All commands should be run in Anaconda Prompt (installed with Miniconda) or PowerShell.
    
    **Microsoft Visual C++ 14.0:**
    
    Some packages require Visual C++ build tools. Download from:
    https://visualstudio.microsoft.com/visual-cpp-build-tools/
    
    **Path separator differences:**
    
    In Python code, use forward slashes or raw strings:
    ```python
    # Good
    path = "Training_Batch_Files/"
    # or
    path = r"Training_Batch_Files\\"
    ```
    
    **Running in production:**
    
    For production on Windows, use `waitress` instead of `gunicorn`:
    ```bash
    pip install waitress
    waitress-serve --port=5001 main:app
    ```
  </Tab>
</Tabs>

## Troubleshooting Common Issues

<AccordionGroup>
  <Accordion title="Error: 'pip' command not found">
    **Problem:** Pip is not installed or not in PATH.
    
    **Solution:**
    ```bash
    # Reinstall pip
    conda install pip
    
    # or use conda's pip explicitly
    /path/to/conda/envs/fraud-detection/bin/pip install -r requirements.txt
    ```
  </Accordion>

  <Accordion title="Error installing XGBoost">
    **Problem:** XGBoost compilation fails on your platform.
    
    **Solution 1 - Use conda:**
    ```bash
    conda install -c conda-forge xgboost=0.90
    ```
    
    **Solution 2 - Install build tools:**
    
    <CodeGroup>
    ```bash Linux
    sudo apt-get install build-essential cmake
    pip install xgboost==0.90
    ```
    
    ```bash macOS
    brew install cmake gcc
    pip install xgboost==0.90
    ```
    
    ```powershell Windows
    # Install Visual Studio Build Tools
    # Then retry: pip install xgboost==0.90
    ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Error: Failed building wheel for scikit-learn">
    **Problem:** Missing build dependencies for scikit-learn.
    
    **Solution:**
    ```bash
    # Install via conda instead
    conda install scikit-learn=0.22.1
    
    # Then install remaining requirements
    pip install -r requirements.txt
    ```
  </Accordion>

  <Accordion title="ImportError: No module named 'flask'">
    **Problem:** Wrong Python interpreter or environment not activated.
    
    **Solution:**
    ```bash
    # Ensure environment is activated
    conda activate fraud-detection
    
    # Verify which Python is being used
    which python
    # Should show: /path/to/miniconda3/envs/fraud-detection/bin/python
    
    # Reinstall Flask if needed
    pip install Flask==1.1.1
    ```
  </Accordion>

  <Accordion title="Port 5001 already in use">
    **Problem:** Another application is using port 5001.
    
    **Solution 1 - Use different port:**
    ```bash
    export PORT=5002
    python main.py
    ```
    
    **Solution 2 - Find and kill process:**
    
    <CodeGroup>
    ```bash Linux/macOS
    # Find process using port 5001
    lsof -i :5001
    
    # Kill the process
    kill -9 <PID>
    ```
    
    ```powershell Windows
    # Find process
    netstat -ano | findstr :5001
    
    # Kill process
    taskkill /PID <PID> /F
    ```
    </CodeGroup>
  </Accordion>

  <Accordion title="SSL Certificate errors">
    **Problem:** Certificate verification fails during package installation.
    
    **Solution:**
    ```bash
    # Update certificates
    conda update certifi
    
    # or install with pip
    pip install --trusted-host pypi.org --trusted-host files.pythonhosted.org -r requirements.txt
    ```
  </Accordion>

  <Accordion title="Permission denied errors">
    **Problem:** Insufficient permissions to create directories or write files.
    
    **Solution:**
    
    **Linux/macOS:**
    ```bash
    # Fix directory permissions
    chmod -R u+w .
    
    # Don't use sudo with conda/pip
    ```
    
    **Windows:**
    ```powershell
    # Run Anaconda Prompt as Administrator
    # or adjust folder permissions in Properties
    ```
  </Accordion>

  <Accordion title="Memory errors during training">
    **Problem:** Insufficient RAM for large datasets.
    
    **Solution:**
    
    1. **Reduce batch size** - Process smaller chunks of data
    2. **Increase swap space** (Linux):
       ```bash
       sudo fallocate -l 4G /swapfile
       sudo chmod 600 /swapfile
       sudo mkswap /swapfile
       sudo swapon /swapfile
       ```
    3. **Use a machine with more RAM**
    4. **Optimize data preprocessing** - Remove unnecessary columns earlier
  </Accordion>
</AccordionGroup>

## Production Deployment

<Warning>
  The Flask development server (`python main.py`) is not suitable for production use. Use a production WSGI server instead.
</Warning>

### Using Gunicorn (Linux/macOS)

Gunicorn is already included in `requirements.txt`.

```bash
# Run with 4 worker processes
gunicorn -w 4 -b 0.0.0.0:5001 main:app

# With automatic restart on code changes
gunicorn -w 4 -b 0.0.0.0:5001 --reload main:app

# Production configuration
gunicorn -w 4 \
  -b 0.0.0.0:5001 \
  --timeout 300 \
  --access-logfile logs/access.log \
  --error-logfile logs/error.log \
  main:app
```

### Using Waitress (Windows)

```bash
pip install waitress
waitress-serve --port=5001 --threads=4 main:app
```

### Environment Variables

Set these environment variables for production:

```bash
export FLASK_ENV=production
export PORT=5001
export WORKERS=4
```

## Uninstallation

To completely remove the fraud detection system:

```bash
# Deactivate environment
conda deactivate

# Remove conda environment
conda env remove -n fraud-detection

# Remove source code
rm -rf /path/to/source

# Optional: Remove Miniconda
rm -rf ~/miniconda3
```

3. **Test with sample data** - Use the provided sample files to isolate the issue
4. **Review dependencies** - Run `pip list` to check installed package versions
